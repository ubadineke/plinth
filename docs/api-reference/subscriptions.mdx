## POST /v1/subscriptions

Create a subscription for a customer.

<CodeGroup>
```bash cURL
curl -X POST https://api.useplinth.xyz/v1/subscriptions \
  -H "Authorization: Bearer sk_test_DOCS_SHARED_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: sub-acme-pro-jun26" \
  -d '{
    "customer_id": "cus_01JXYZ...",
    "plan_id": "plan_pro_01...",
    "payment_method": "card",
    "payment_method_id": "tok_test_visa"
  }'
```

```typescript Node.js
const subscription = await client.subscriptions.create({
  customerId: 'cus_01JXYZ...',
  planId: 'plan_pro_01...',
  paymentMethod: 'card',
  paymentMethodId: 'tok_test_visa',
}, { idempotencyKey: 'sub-acme-pro-jun26' });
```
</CodeGroup>

**Request body**

<ParamField body="customer_id" type="string" required>
  The customer to subscribe.
</ParamField>

<ParamField body="plan_id" type="string" required>
  The plan to subscribe the customer to.
</ParamField>

<ParamField body="payment_method" type="string" required>
  Payment rail. Values: `card`, `transfer`.
</ParamField>

<ParamField body="payment_method_id" type="string">
  Card token (required if `payment_method` is `card`). Not used for `transfer`.
</ParamField>

<ParamField body="billing_mode" type="string">
  `advance` (default) or `arrears`. See [arrears billing guide](/guides/arrears-billing).
</ParamField>

<ParamField body="start_at" type="string">
  ISO 8601 date to start the subscription. Defaults to now.
</ParamField>

## GET /v1/subscriptions/:id

Retrieve a subscription.

```bash
curl https://api.useplinth.xyz/v1/subscriptions/sub_01JABC... \
  -H "Authorization: Bearer sk_test_DOCS_SHARED_KEY"
```

```json Response
{
  "id": "sub_01JABC...",
  "customer_id": "cus_01JXYZ...",
  "plan_id": "plan_pro_01...",
  "state": "active",
  "payment_method": "card",
  "billing_mode": "advance",
  "current_period_start": "2026-06-01T00:00:00Z",
  "current_period_end": "2026-07-01T00:00:00Z",
  "next_bill_at": "2026-07-01T00:00:00Z",
  "trial_end": null,
  "created_at": "2026-06-01T00:00:00Z"
}
```

## GET /v1/subscriptions/:id/status

Get a quick status summary — state, next billing date, and how long the subscription has been in the current state.

```bash
curl https://api.useplinth.xyz/v1/subscriptions/sub_01JABC.../status \
  -H "Authorization: Bearer sk_test_DOCS_SHARED_KEY"
```

```json Response

  "state": "past_due",
  "next_bill_at": "2026-07-01T00:00:00Z",
  "days_in_state": 3,
  "grace_expires_at": null,
  "next_retry_at": "2026-06-04T00:00:00Z"
}
```

## POST /v1/subscriptions/:id/cancel

Cancel a subscription. **No body** — the behaviour follows the tenant's `cancel_policy`:

- **`end_of_period`** (default) — the subscription stays **active** (access retained) and won't renew;
  it moves to `canceled` at the next renewal tick. Any pending plan change is dropped.
- **`immediate`** — the subscription moves to `canceled` now; access ends immediately.

```bash cURL
curl -X POST https://api.useplinth.xyz/v1/subscriptions/sub_01JABC.../cancel \
  -H "Authorization: Bearer sk_test_DOCS_SHARED_KEY"
```

```json Response
{
  "object": "subscription",
  "subscriptionId": "sub_01JABC...",
  "state": "active",
  "cancelAtPeriodEnd": true,
  "effectiveAt": "2026-08-01T00:00:00Z",
  "policy": "end_of_period"
}
```

<ResponseField name="cancelAtPeriodEnd" type="boolean">
  `true` when the subscription is canceling at period end (still active until `effectiveAt`).
  `false` on an immediate cancel.
</ResponseField>
<ResponseField name="effectiveAt" type="string">
  When access actually ends — `now` for an immediate cancel, or the current period end for
  `end_of_period`.
</ResponseField>

## POST /v1/subscriptions/:id/reactivate

Undo a still-pending `end_of_period` cancel — the subscription keeps renewing. No body. Only valid
while the subscription is still active (before it has actually ended); on an already-`canceled`
subscription this returns `409` — start a new subscription instead.

```bash cURL
curl -X POST https://api.useplinth.xyz/v1/subscriptions/sub_01JABC.../reactivate \
  -H "Authorization: Bearer sk_test_DOCS_SHARED_KEY"
```

```json Response
{
  "object": "subscription",
  "subscriptionId": "sub_01JABC...",
  "state": "active",
  "cancelAtPeriodEnd": false,
  "effectiveAt": "2026-08-01T00:00:00Z",
  "policy": "n/a"
}
```
